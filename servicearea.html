Service Area Analysis
Short Description
Indetifies the reachable, catchment area around facilities (ex : hospitals, stores) based on street network travel time or distance. The differentiate from simple raduis buffers in their method that calculates accessibility along actual, traversable roads, producing polygons or lines that illustrate service coverage.
In this map, the service area are categorized into six class coverage 300, 600, 900, 1200, 2400, 8000 in meters. Service area coverage in percent shows the fifth class is the highest percentage with almost 40%. the lowest percentage wit 2,63% showed with the fist coverage category (0-300m). Overall, hospital service area has covered with average 20%.
Data
Method : Network Analysis
Tools : ArcMap
  1. Road Shapefile
  2. Impedance data like travel time (minutes) or distance
  3. Stop point or visited place (in this map is hospitals)
  4. Barrier (optional) represent as a roadblock to find an alternate route
Deficiencies and possible improvements
  1. One drawback of this map is that it ignores the road types. The analysis is based solely on the shortest road length, but in reality, some types of roads may be impassable by vehicles.
  2. road inspections and restrictions will be required if they are to be used as detailed information or as a reference for decision-making.
  3. Restrictions on attributes such as specific road types can be included in the "Restrictions" section.
  4. While creating this map, I encountered several issues with the road data. The road data I had was incomplete in several areas, so I had to remove two hospital points to avoid this error. Therefore, this analysis relies heavily on the level of detail and accuracy of the road data used.
